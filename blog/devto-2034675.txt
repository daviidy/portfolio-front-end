13:T2800,<p>Deploying a Next.js app to GitHub Pages can be a bit tricky due to the static nature of GitHub Pages and the dynamic features of Next.js. In this article, I'll walk you through the steps to successfully deploy do it.</p>
<p><strong>Prerequisites</strong></p>
<ul>
<li>A GitHub account</li>
<li>Node.js and npm installed on your machine</li>
<li>A Next.js project ready to deploy</li>
</ul>
<p><strong>Step 1: Install gh-pages</strong></p>
<p><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark github-light" style="--shiki-dark:#e1e4e8;--shiki-light:#24292e;--shiki-dark-bg:#24292e;--shiki-light-bg:#fff"><span data-line=""><span>npm install gh-pages --save-dev</span></span></code></span></p>
<p><strong>Step 2: Update next.config.mjs</strong></p>
<p>Next, you need to update your next.config.js (or next.config.mjs) file to handle the base path and asset prefix correctly.</p>

<span data-line=""><span style="--shiki-dark:#F97583;--shiki-light:#D73A49">const</span><span style="--shiki-dark:#79B8FF;--shiki-light:#005CC5"> nextConfig</span><span style="--shiki-dark:#F97583;--shiki-light:#D73A49"> =</span><span style="--shiki-dark:#E1E4E8;--shiki-light:#24292E"> {</span></span>

<span data-line=""><span style="--shiki-dark:#E1E4E8;--shiki-light:#24292E">  images: {</span></span>

<span data-line=""><span style="--shiki-dark:#E1E4E8;--shiki-light:#24292E">};</span></span>

<span data-line=""><span style="--shiki-dark:#F97583;--shiki-light:#D73A49">export</span><span style="--shiki-dark:#F97583;--shiki-light:#D73A49"> default</span><span style="--shiki-dark:#E1E4E8;--shiki-light:#24292E"> nextConfig;</span></span></code></pre></figure>
<p><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark github-light" style="--shiki-dark:#e1e4e8;--shiki-light:#24292e;--shiki-dark-bg:#24292e;--shiki-light-bg:#fff"><span data-line=""><span>isProd</span></span></code></span> checks if the NODE_ENV environment variable is set to 'production'. <p><strong>Step 3: Update package.json</strong></p>
<p>Update your package.json to include the homepage field and the deploy scripts.</p>

<p><strong>Step 4: Deploy Your App</strong></p>
<p>Run <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark github-light" style="--shiki-dark:#e1e4e8;--shiki-light:#24292e;--shiki-dark-bg:#24292e;--shiki-light-bg:#fff"><span data-line=""><span>npm run predeploy</span></span></code></span> and <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark github-light" style="--shiki-dark:#e1e4e8;--shiki-light:#24292e;--shiki-dark-bg:#24292e;--shiki-light-bg:#fff"><span data-line=""><span>npm run deploy</span></span></code></span></p>
<p>These commands will:</p>
<ul>
<li>Build your project.</li>
<li>Export it to the out directory.</li>
<li>Deploy it to the gh-pages branch on GitHub.
Push your work to your GitHub branch before the next step</li>
</ul>
<p><strong>Step 5: Configure GitHub Pages</strong></p>
<p>Go to your repository on GitHub.
Navigate to Settings > Pages.
Under Source, select the gh-pages branch.
Save the settings.
Add manually the .nojekyll file at the root of the gh-pages branch on GitHub. Learn more about this <a href="https://github.com/vercel/next.js/discussions/33751">here</a>.</p>
<p><strong>Step 6: Verify Deployment</strong></p>
<p>After deploying, open your GitHub Pages URL (e.g., https://.github.io/) to verify that your app is live.</p>
